## Configuring Django for PostgreSQL and Neo4j Multiple Databases

Django supports hybrid setups with multiple databases, where relational data (e.g., JSON in PostgreSQL) uses the ORM, and graph data (Neo4j via django-neomodel) uses neomodel's OGM. Your current settings.py has an empty "default" database and "abstract" for Postgres, plus neo4j config—this works since neomodel doesn't require a DATABASES entry (it uses `NEOMODEL_NEO4J_BOLT_URL` independently). To manage both properly, set "default" to Postgres for auth/sessions (required for Django core), keep "abstract" for semantic JSON (e.g., vector embeddings or abstracted paper metadata from your RAG projects), and configure Neo4j separately. Use routers for routing if needed, but neomodel models auto-route to Neo4j without them.[1][2][3][4]

### Key Changes Explained

- **"default" Database**: Now a full Postgres config—Django requires this for auth/sessions. It shares the "biosphere" DB but defaults to main schema. Migrate core apps here: `python manage.py migrate`.[2]
- **"abstract" Database**: Kept for semantic JSON (e.g., store paper abstracts as JSONB with embeddings). Use separate schema (`search_path=semantic`) to isolate from "default"—create schema in Postgres: `CREATE SCHEMA semantic;`. This avoids table conflicts in shared DB.[5][2]
- **Neo4j Setup**: Use `config.DATABASE_URL` (modern neomodel) alongside the old `NEOMODEL_NEO4J_BOLT_URL` for compatibility. django-neomodel models (e.g., `StructuredNode`) auto-connect to Neo4j without touching DATABASES. Test: Add a neomodel model and run `python manage.py shell` → `from yourapp.models import Paper; Paper.nodes.all()`.[3][4][6]
- **Routers (Optional)**: For Postgres models in "abstract", create a router (below) to route them automatically. Neo4j doesn't need this—neomodel handles routing internally. Add to `INSTALLED_APPS` if using custom auth.[1][5]
- **Security/Prod**: Use env vars (e.g., `os.environ.get('DB_PASSWORD')`) for secrets. For Docker/Podman (your setup), use volumes for Postgres/Neo4j persistence.[2]

### Using Databases Combinatorily in Your Django App

In your "biosphere" app, define separate models for each DB. Use transactions/views to combine: Query Postgres for JSON metadata, then traverse Neo4j for relations (e.g., paper citations in your space biology project). Neomodel integrates seamlessly with Django views/serializers.[7][8][1]

#### 1. Models Setup (in biosphere/models.py)

```python
# Postgres models (use "abstract" DB)
from django.db import models
from django.db.models.signals import post_save
from django.dispatch import receiver
import json

class SemanticPaper(models.Model):  # For JSON storage in Postgres
    doi = models.CharField(max_length=255, unique=True)
    semantic_data = models.JSONField(default=dict)  # Store embeddings, topics as JSON
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        db_table = 'semantic_papers'  # Table in "abstract" schema

    def save(self, *args, **kwargs):
        # Optional: Validate JSON or add embeddings
        if isinstance(self.semantic_data, str):
            self.semantic_data = json.loads(self.semantic_data)
        super().save(*args, **kwargs)

# Neo4j models (via django-neomodel/neomodel)
from neomodel import StructuredNode, StringProperty, RelationshipTo

class PaperNode(StructuredNode):  # For graph relations
    doi = StringProperty(unique_index=True)
    title = StringProperty()
    abstract = StringProperty()

    # Relationships (from your earlier schema)
    authors = RelationshipTo("Author", "AUTHORED_BY")
    cites = RelationshipTo("PaperNode", "CITES")

class Author(StructuredNode):
    name = StringProperty(unique_index=True)
    # etc.
```

- **Postgres (JSON)**: Use JSONField for flexible semantic storage (e.g., {"topics": ["microgravity"], "embeddings": [0.1, 0.2]}). Query: `SemanticPaper.objects.using('abstract').filter(doi='10.1016/j.cell.2014.07.025')`.[2]
- **Neo4j (Graph)**: Define as `StructuredNode`—auto-saves to Neo4j. Query: `PaperNode.nodes.get_or_none(doi='...')`. For your RAG, store raw metadata in Postgres, relations in Neo4j.[4][6]

#### 2. Database Router (biosphere/routers.py) - For Postgres Routing

If you want automatic routing for "abstract" models (e.g., app_label='biosphere' with `using='abstract'`):

```python
class AbstractRouter:
    def db_for_read(self, model, **hints):
        if model._meta.app_label == 'biosphere' and hasattr(model, '_meta_db_table'):  # Custom attr for JSON models
            return 'abstract'
        return 'default'  # Or None for no routing

    def db_for_write(self, model, **hints):
        if model._meta.app_label == 'biosphere' and hasattr(model, '_meta_db_table'):
            return 'abstract'
        return 'default'

    def allow_migrate(self, db, app_label, model_name=None, **hints):
        if app_label == 'biosphere' and db == 'abstract':
            return True  # Migrate JSON models to "abstract"
        elif db == 'default':
            return True  # Core to default
        return False  # Neo4j handled separately
```

In models.py for Postgres models: Add `class Meta: using = 'abstract'` or route via app_label. Migrations: `python manage.py makemigrations biosphere --database=abstract`.[5]

#### 3. Combining in Views/Services (biosphere/views.py or services.py)

Fetch from both DBs in a view—e.g., API endpoint for paper details:

```python
from django.http import JsonResponse
from django.views import View
from .models import SemanticPaper, PaperNode

class PaperDetailView(View):
    def get(self, request, doi):
        # Postgres: Get semantic JSON
        try:
            paper_json = SemanticPaper.objects.using('abstract').get(doi=doi)
            semantic_data = paper_json.semantic_data
        except SemanticPaper.DoesNotExist:
            semantic_data = {}

        # Neo4j: Get graph relations
        try:
            paper_node = PaperNode.nodes.get(doi=doi)
            authors = [a.name for a in paper_node.authors.all()]
            cited_papers = [p.title for p in paper_node.cites.all()]
        except PaperNode.DoesNotExist:
            authors = cited_papers = []

        # Combine
        combined = {
            'doi': doi,
            'semantic': semantic_data,
            'graph': {
                'authors': authors,
                'citations': cited_papers,
            }
        }
        return JsonResponse(combined)
```

- **Transactions**: For atomic ops, use `with transaction.atomic(using='default'):` for Postgres; neomodel has `PaperNode.save()` with auto-transaction. For cross-DB, manual rollback or Celery tasks.[7][1]
- **RAG Integration**: In your biosphere app, query Postgres for embeddings (e.g., via pgvector extension: `pip install psycopg2`), then Neo4j for paths (e.g., `paper.cites.all()` for citation chains). Serialize to JSON for FastAPI/Next.js frontend.[8][9]
- **Admin/Shell**: Neo4j admin via django-neomodel (experimental); test in shell: `python manage.py shell --database=abstract` for Postgres. For Neo4j: Direct in shell.[10][11]

### Setup and Testing Steps

1. **DB Init**: Start Postgres/Neo4j (e.g., via Podman: `podman run -p 5432:5432 postgres`). Create DB/schema: `createdb biosphere; psql -d biosphere -c "CREATE SCHEMA semantic;"`.
2. **Migrations**: `python manage.py makemigrations --database=default`; `python manage.py migrate --database=default`; For abstract: `python manage.py makemigrations biosphere --database=abstract && python manage.py migrate --database=abstract`.
3. **Test Queries**: In shell: Create Postgres entry, then Neo4j node, link via DOI. Query combined as above.
4. **Prod Tips**: Use separate Postgres DBs for isolation; scale Neo4j with clusters for 600+ papers. Monitor with Django debug toolbar or neomodel logging. For your GraphRAG, add vector search in Postgres alongside Neo4j traversals.[12][13][2]

This setup leverages your Django/Neo4j experience for a robust RAG backend. If errors (e.g., connection), check firewall/credentials.[14][1]
